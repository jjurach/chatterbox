# Task 6.14: Unified Configuration System

**Date:** 2026-03-25 11:36:51
**Status:** Completed
**Epic:** Epic 6 — Backend Deployment & HA Connection (Tasks 6.14-6.21)

## Objective

Implement a unified configuration system that makes `~/.config/chatterbox/settings.json` the single source of truth for all Chatterbox and Mellona configuration. Add API key support with automatic generation and persistence.

## Changes Implemented

### 1. Enhanced Settings Class (`src/chatterbox/config/__init__.py`)

#### New Features:
- **Custom JSON Settings Source:** Added `ChatterboxJsonSettingsSource` class that reads from `~/.config/chatterbox/settings.json`
- **API Key Field:** Added `api_key: str | None` field that defaults to None
- **Auto-Generation:** Implemented `ensure_api_key()` method that:
  - Generates a UUID4 if api_key is None
  - Persists the key to `~/.config/chatterbox/settings.json`
  - Logs the generated key at INFO level: `"API key (auto-generated): <key>"`
  - Creates the config directory if it doesn't exist
  - Handles file write errors gracefully with warnings
- **Nested Config Flattening:** Added `_flatten_nested_settings()` helper to extract nested JSON values:
  - `api.key` → `api_key`
  - `memory.conversation_window_size` → `conversation_window_size`
  - `logging.level` → `log_level`

#### Configuration Priority (Highest to Lowest):
1. Environment variables (`CHATTERBOX_*`)
2. `~/.config/chatterbox/settings.json`
3. `.env` file (if present)
4. Built-in defaults

#### Mellona Integration:
- `get_mellona_config_path()` now prefers `~/.config/chatterbox/settings.json` if it exists
- Falls back to packaged `mellona.yaml` for backward compatibility
- Mellona is initialized via `get_config(config_chain=[str(settings_path)])`

### 2. Configuration File (`~/.config/chatterbox/settings.json`)

Created a comprehensive schema supporting:

```json
{
  "server": { "host", "port" },
  "api": { "key", "comment" },
  "conversation": { "port", "host" },
  "providers": {
    "faster_whisper": { "model", "device", "language" },
    "piper": { "voice", "sample_rate" },
    "ollama": { "base_url" }
  },
  "profiles": {
    "default": { "provider", "model", "temperature" }
  },
  "stt_profiles": { "default": { "provider" } },
  "tts_profiles": { "default": { "provider" } },
  "memory": { "conversation_window_size" },
  "logging": { "level" }
}
```

Extra fields are ignored, allowing future expansion without code changes.

## Notes

### Configuration Directory Creation

The system automatically creates `~/.config/chatterbox/` on first API key generation. For manual setup:

```bash
mkdir -p ~/.config/chatterbox
```

### Backward Compatibility

- Packaged `mellona.yaml` is still supported for systems without `~/.config/chatterbox/settings.json`
- `mellona_config_path` env var still takes precedence if explicitly set
- All existing code paths continue to work unchanged

## Implementation Notes

1. **JSON Settings Source:** Uses pydantic-settings' `JsonConfigSettingsSource` with custom override to read from user config directory
2. **Error Handling:** Invalid JSON files log warnings but don't crash; system falls back to defaults
3. **Nested Flattening:** Handles both direct and nested field access (e.g., `api.key` → `api_key`)
4. **Thread Safety:** File I/O for persistence is blocking; suitable for startup initialization
